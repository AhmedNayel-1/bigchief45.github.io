Python Serverless OCR Microservice

A serverless microservice written in Python and deployed in AWS Lambda that uses Google's Tesseract OCR library to extract OCR (Optical Character Recognition) data from book pages image files that are uploaded to an S3 bucket as image files.

The OCR data is extracted in hOCR format and then reformatted as JSON to be saved in an S3 bucket. This data OCR data can then be used to simulate text highlighting in an image-based online reader like Google's or Amazon's.

Librarian: Python Serverless PDF Processing Microservices

A collection of serverless microservices written in Python and deployed in AWS Lambda. One of these services extracts meta data, table of contents, and full text from PDF files. This data is then stored in different locations. For example, full text of PDFs is stored in Elasticsearch so that full text search can be made on all uploaded PDF files in the cloud

Another service is in charge of converting every page of the PDF into an image. So that each page can be displayed by an online reader as an image. When an image is requested, another service that leverages Amazon API Gateway is in charge of fetching the image from a CDN and providing its corresponding OCR data.

eLibro Content Manager

eLibro Content Manager is a native desktop Java application built using the Netbeans Rich Client Platform (RCP), the platform that powers the Netbeans IDE.

The application uses the AWS Java SDK to allow interaction with files and data stored in Amazon S3 and Amazon DynamoDB. This application allows employees in charge of uploading the company's content (books) to upload and manipulate content files to S3, so that they can automatically be processed by Librarian.

Additionally, it allows viewing the current health status of each book, depending on whether the book was successfully processed or not.
